# WebRTC P2P Demo - 快速开始

## 前置条件

确保你已经安装了依赖：
```bash
npm install
```

## 启动步骤

### 1. 启动信令服务器

打开一个终端，运行：

```bash
npm run signaling
```

你应该看到：
```
✅ WebRTC 信令服务器运行在 http://localhost:3002
   等待 P2P 连接建立...
```

### 2. 启动 Next.js 开发服务器

打开另一个终端，运行：

```bash
npm run dev
```

你应该看到 Next.js 启动成功的消息。

### 3. 打开 Demo 页面

在浏览器中打开：
```
http://localhost:3000/webrtc-demo
```

## 测试流程

### 步骤 1：输入房间号

在页面的输入框中输入一个房间号，例如：`123456`

### 步骤 2：加入房间

1. 点击 **"Peer 1 加入"** 按钮
   - 这模拟提词器服务端（发起者）
   - 日志应该显示连接过程

2. 点击 **"Peer 2 加入"** 按钮
   - 这模拟遥控器客户端（接收者）
   - 当两个 Peer 都加入后，应该开始建立 P2P 连接

### 步骤 3：等待连接建立

观察连接状态面板：
- Peer 1 和 Peer 2 的状态应该从 "❌ 未连接" 变为 "✅ 已连接"
- 日志中会显示详细的连接过程

### 步骤 4：测试消息发送

1. 在消息输入框中输入测试消息
2. 点击 **"Peer 1 发送"** 或 **"Peer 2 发送"**
3. 观察日志中的消息收发情况

## 故障排查

### 问题 1：信令服务器连接失败

**症状**：日志显示 "❌ 信令服务器连接失败"

**解决方案**：
- 检查信令服务器是否成功启动（端口 3002）
- 检查是否有防火墙阻止连接
- 尝试访问 http://localhost:3002 看是否有响应

### 问题 2：P2P 连接失败

**症状**：两个 Peer 都加入了房间，但 P2P 连接一直不成功

**可能原因**：
- NAT 穿透失败（需要 TURN 服务器）
- 浏览器不支持 WebRTC
- HTTPS 问题（生产环境需要 HTTPS）

**解决方案**：
- 确保使用支持 WebRTC 的现代浏览器（Chrome、Firefox、Safari）
- 本地测试使用 localhost 不受影响
- 生产环境需要配置 TURN 服务器

### 问题 3：消息发送失败

**症状**：点击发送按钮后显示 "❌ 发送失败"

**解决方案**：
- 检查 P2P 连接是否已建立（状态显示 "✅ 已连接"）
- 检查数据通道是否打开
- 查看浏览器控制台是否有错误

## 技术细节

### 信令服务器

- 使用 Socket.IO 进行信令交换
- 处理房间管理、offer/answer/ICE candidate 转发
- 端口：3002

### WebRTC 连接

- 使用 RTCPeerConnection 建立 P2P 连接
- 使用 RTCDataChannel 进行数据通信
- STUN 服务器：Google 公共 STUN 服务器

### 数据流

```
Peer 1 → 信令服务器 → Peer 2 (建立连接)
    ↓
P2P 连接建立完成
    ↓
Peer 1 ←――――――――――――→ Peer 2 (直接通信)
```
